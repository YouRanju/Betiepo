필요한 DB

1. 유저정보저장 : userTbl
유저 이름, 유저 이메일, 유저 비밀번호 (여기까지는 NOT NULL), 유저사진, 유저 영문이름, 전화번호, 포부 저장
이메일 PK
사용하는 곳 : 로그인(select), 회원가입(insert), 마이페이지(update)

2. 유저 포트폴리오 리스트 저장 : userPortfolioList
유저 이메일, 포트폴리오 이름, 대표 사진, 카테고리, 날짜, 설명, 즐겨찾기 저장
이메일 FK, 포트폴리오 이름,대표사진을 uk로 하라는데 나는 잘... 카테고리 FK
(내가 곰곰히 생각했는데 대표사진 저장할 때 두가지 방법이 있는데 하나는 바이너리? 그걸로 저장, 두번째는 주소저장인데
쉬운건 주소 저장이고 쓸만한건 바이너리 그거임)
사용하는곳 : 인덱스 페이지(로그인 후 바로 그곳, select, insert)
기본으로 하나 저장하셈 (예제)

3. 포트폴리오 : portfolioContent
유저 이메일, 포트폴리오 이름, 장르, 시작날짜, 끝난날짜, 담당, 깃허브 주소, 비고, 대표사진, 설명, 목적, 목표사용자, 시나리오, 핵심포인트, 완성이미지, 공부방향, 어려웠던점, 총 소감 저장 (실질적인 저장하는 곳)
이메일, 포트폴리오 이름, 대표사진 FK
기본으로 하나 저장하셈 (예제)
사용하는곳 : 포트폴리오 작성 페이지 (select, insert, update)

4. 카테고리저장
유저 이메일, 카테고리
이메일 FK, 카테고리 PK
이거 기본으로 두 개 저장해놓으셈 (프로젝트, 외부활동) - 플러스 모양있는 인덱스 누르면 추가됨(insert)
사용하는 곳 인덱스 페이지에 import 해놓음(select, insert, delete)
